Mobile Architecture and Programming

Inventory Tracking App (Android / Java – InvenTrack)

Mobile application that helps users sign in and manage item quantities with a simple, accessible interface.

What I did

  • Built login / registration backed by SQLite with EncryptedSharedPreferences for secure session handling.
  • Added password policy and PBKDF2-hashed credentials with per-user salt.
  • Implemented role-based access: the first registered user becomes admin; admins can add, update, delete, and send SMS alerts; regular users are read-only.
  • Stored and updated items locally in a grid-style inventory with search, add, edit, and delete actions.
  • Added SKU-based validation (max 12 digits, unique index), quantity limits, and input checks to prevent crashes.
  • Added “Remember username” option and improved login error handling.
  • Built an SMS alert settings screen with runtime permission handling, “Don’t ask again” flow, and customizable templates.
  • Polished UI with branded InvenTrack theme, orange Material buttons, logo on login screen, and better accessibility.

Planned enhancements

  • Expand test coverage with unit and UI tests using Espresso/JUnit for more reliable releases.
  • Add cloud sync option so admins can back up inventory to a remote database.
  • Introduce analytics to log low-stock events for reporting.
  • Tech: Java, Android API 34, SQLite / Room, EncryptedSharedPreferences, PBKDF2
  • Skills: Mobile UX, secure sign-in, MVVM architecture, role-based access, testing

Data Structures and Algorithms

Course Advising Program (C++)

Command-line tool that organizes course information and makes it easy to list classes and view details.

What I did

  • Loaded course data from a CSV file into an in-memory Binary Search Tree keyed by course ID.
  • Implemented iterative in-order traversal to print an alphanumerically sorted list of courses.
  • Displayed course details (ID, name, prerequisites) for a selected course.
  • Added error handling for malformed rows and verified input integrity during loads.
  • Enhanced traversal performance by combining a BST lookup with an unordered_map index for O(1) search.
  • Documented runtime complexity (Big-O) for each operation in the project report.

Planned enhancements

  • Add a direct-lookup table to speed up repeated queries in very large catalogs.
  • Expand cycle-detection in prerequisites to reject invalid course plans.
  • Add unit tests for insert/search/remove operations to prevent regressions.
  • Tech: C++17
  • Skills: Algorithms, data organization, iterative traversal, error handling, Big-O analysis

Client-Server Development

MongoDB Dashboard (Python / Dash – Grazioso Salvare)

Web dashboard that authenticates users, provides CRUD workflows, visualizes outcomes on tables, charts, and maps, and demonstrates secure database practices.

What I did

  • Connected securely to MongoDB Atlas using TLS-validated user login and read-only runtime role (RO).
  • Built interactive Dash dashboard with radio-button filters, DataTable, map fallback to Austin, and bar charts for outcomes.
  • Added Admin Tools panel with schema and index viewers (RO-safe callbacks).
  • Configured JSON Schema validation for aac.animals collection and verified expected rejection on bad input.
  • Proved RBAC enforcement: RO user denied inserts/updates/deletes at runtime.
  • Customized branding with assets/grazioso_logo.png, light layout styling, and external port 8052 launch in JupyterLab.

Planned enhancements

  • Expose a small REST API for external analytics clients.
  • Containerize with Docker + HTTPS for secure deployment.
  • Add observability metrics (timing, query counts) to monitor dashboard health.
  • Create compound indexes and server-side pagination for smoother performance on large data sets.
  • Tech: Python 3.12, Dash, MongoDB Atlas (TLS)
  • Skills: API design, schema validation, RBAC security, data visualization, secure deployment
