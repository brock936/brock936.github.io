Mobile Architecture and Programming

Inventory Tracking App (Android/Java)

Mobile application that helps users sign in and manage item quantities with a simple, accessible interface.

What I did

  • Added a login screen backed by SQLite and basic session handling.
  • Stored and updated items locally; showed inventory in a grid with add, edit, and delete actions.
  • Implemented input checks to prevent bad entries and crashes.
  • Sent low stock SMS notifications after the user granted permission.

Planned enhancements

  • Refactor to MVVM with a Repository and Room for the data layer to improve maintainability.
  • Add secure sign-in with hashed credentials and encrypted local storage.
  • Introduce role based permissions so only admins can delete items.
  • Polish UI with Material patterns and stronger accessibility support.
  • Add unit and UI tests with continuous integration for reliable releases.
  • Tech: Java, Android, SQLite/Room
  • Skills: Software engineering, mobile UX, testing, security

Data Structures and Algorithms

Course Advising Program (C++)

Command-line tool that organizes course information and makes it easy to list classes and view details.

What I did

  • Loaded course data from a CSV file.
  • Stored courses in a Binary Search Tree keyed by course ID.
  • Printed an alphanumeric list of courses using in order traversal.
  • Displayed details for a selected course, including title and prerequisites.

Planned enhancements

  • Add a direct lookup table in addition to the tree so finding a course by ID is instant.
  • Replace recursive listings with an iterative version to handle larger catalogs safely.
  • Model prerequisites as a small graph and check for cycles to prevent invalid plans.
  • Validate inputs from the CSV and report errors clearly to avoid bad data.
  • Write focused tests and small timing checks to show real improvements.
  • Tech: C++
  • Skills: Problem solving, data organization, planning logic

Client/Server Development

MongoDB Dashboard (Python/Dash)

Web dashboard that authenticates users, provides create read update and delete operations, and visualizes results for faster decisions.

What I did

  • Connected to MongoDB collections and loaded data into interactive tables and charts.
  • Built authenticated CRUD workflows with filtering and search.
  • Used aggregation queries to present summaries and trends.

Planned enhancements

  • Add role based access so permissions match each user’s responsibilities.
  • Create compound indexes and server side pagination for smoother performance on larger data sets.
  • Expose a small REST API and validate inputs to keep data clean.
  • Containerize with Docker and enable HTTPS to support secure deployment.
  • Add basic observability such as timing and query counts to monitor health.
  • Tech: Python, Dash, MongoDB
  • Skills: Databases, data visualization, API design, deployment
